# Bastard Bonds 简体中文化工具指南

这个工具集用于将Bastard Bonds游戏进行简体中文化处理。

**注意**

1. 内置中文文本，使用AI翻译  
2. 存在BUG，部分翻译文本没有正确的显示  
3. 图片内文本未翻译，详见Data\Unity_Assets_Files

## 使用方法
1. 在游戏根目录（BB.exe所在的地方）放置BastardBonds_ZH文件夹
2. UnityEX.exe置入UnityEX文件夹
3. 运行0_一键本地化.bat



## 文件构成

- `0_一键本地化.bat` - 一键执行完整本地化流程
- `1_备份游戏文件.bat` - 创建游戏文件备份
- `2_制作字体文件.bat` - 制作和调整字体文件
- `3_制作游戏文本.bat` - 处理翻译文件
- `4_导入字体&贴图.bat` - 导入字体和贴图资源
- `5_dll文件本地化.bat` - DLL文件本地化处理
- `6_二进制文件本地化.bat` - 二进制文件本地化处理
- `Core/` - PowerShell脚本文件夹
- `UnityEX/` - Unity资源提取和导入工具

## 各批处理文件说明

### 1. 0_一键本地化.bat

**功能**: 一键执行完整的简体中文化流程。

**处理内容**:
1. **检查UnityEX.exe是否存在**
   - 操作Unity资产文件所需的工具
   
2. **恢复备份**
   - 如果备份不存在则自动创建
   - 从现有备份恢复数据
   
3. **处理字体文件**
   - 复制字体文件到指定位置
   - 替换游戏字体资源
   
4. **处理翻译文件**
   - 使用SPDL.ps1处理TSV翻译文件
   - 生成可用于游戏的翻译数据
   
5. **导入字体和贴图资源**
   - 使用UnityEX.exe导入资源文件
   
6. **翻译DLL内部文本**
   - 使用Convert-16LEText.ps1脚本将DLL内部的英文文本转换为中文
   
7. **翻译二进制文本**
   - 使用rebinary.ps1脚本翻译二进制文件中的文本
   - 处理Maps、Dialogue、Core文件夹中的文件

**实现原理**:
- **字体处理**: 复制DDS字体文件并替换游戏资源
- **翻译处理**: SPDL.ps1处理翻译数据转换
- **DLL翻译**: Convert-16LEText.ps1检测UTF-16LE编码的字符串，根据翻译数据进行替换
- **二进制翻译**: rebinary.ps1解析二进制文件，检测可读字符串并进行翻译
- **资产操作**: UnityEX.exe操作Unity资产包文件以更改字体和纹理

### 2. 1_备份游戏文件.bat

**功能**: 创建游戏数据的备份。

**处理内容**:
- 创建`Backup`文件夹（如果已存在则删除后重新创建）
- 将`Assembly-CSharp.dll`复制到Backup文件夹
- 将`Core`、`Dialogue`、`Maps`文件夹复制到Backup文件夹

**使用目的**: 在进行本地化处理前，安全保存原始游戏数据。

### 3. 2_制作字体文件.bat

**功能**: 调整字体大小。

**处理内容**:
- 要求用户输入字体大小（0-5像素）
- 执行`Core/fnt.ps1`脚本来转换字体数据
- 修改后的字体将影响游戏的中文显示质量

**实现原理**:
- `fnt.ps1`解析`.fnt`格式的字体文件
- 调整字体字形的位置信息和大小信息
- 更改会反映在`FONT_XBOX_PRE.font_raw`文件中

### 4. 3_制作游戏文本.bat

**功能**: 处理翻译文件。

**处理内容**:
- 使用SPDL.ps1处理TSV翻译文件
- 生成可用于游戏的翻译数据文件

**实现原理**:
- 使用`SPDL.ps1`脚本
- 处理TSV格式的翻译数据
- 生成dll_zh.tsv和binary_zh.tsv文件

### 5. 4_导入字体&贴图.bat

**功能**: 导入字体和贴图资源到游戏文件。

**处理内容**:
- 检查UnityEX.exe工具是否存在
- 恢复备份文件
- 导入字体资源到游戏文件
- 使用UnityEX导入资源文件

**实现原理**:
- 使用`UnityEX.exe`操作Unity资产包文件
- 导入字体和纹理资源
- 修改游戏资源文件

### 6. 5_dll文件本地化.bat

**功能**: 处理DLL文件中的文本本地化。

**处理内容**:
- 检查UnityEX.exe工具是否存在
- 恢复备份文件
- 处理DLL文件中的文本翻译
- 使用Convert-16LEText.ps1脚本

**实现原理**:
- **DLL翻译**: `Convert-16LEText.ps1`检测UTF-16LE编码的字符串，根据翻译数据进行替换

### 7. 6_二进制文件本地化.bat

**功能**: 处理二进制文件中的文本本地化。

**处理内容**:
- 检查UnityEX.exe工具是否存在
- 恢复备份文件
- 翻译二进制文件中的文本内容
- 处理Maps、Dialogue、Core文件夹中的文件

**实现原理**:
- **二进制翻译**: `rebinary.ps1`解析二进制文件，检测可读字符串并进行翻译

## PowerShell脚本实现原理

### SPDL.ps1
- 处理TSV格式的翻译数据
- 字符编码转换和句读点处理
- 写入输出文件

### Convert-16LEText.ps1
- 搜索二进制文件中的UTF-16LE字符串
- 根据翻译映射替换字符串
- 在字节长度限制内执行翻译（超出则输出错误日志）

### rebinary.ps1
- 直接解析二进制文件
- 检测和翻译可读字符串
- 动态编译C#代码以实现高速处理

### fnt.ps1
- 解析BMFont格式的字体文件
- 调整字形的位置和大小信息
- 转换为游戏使用的二进制字体格式

## 注意事项

1. **风险自负**: 这个工具会修改游戏数据，使用风险自负
2. **必须备份**: 执行本地化前请务必创建备份
3. **兼容性**: 游戏版本更新可能导致工具无法正常工作
4. **字体调整**: 如果字体显示异常，尝试调整字体大小

## 使用步骤

1. 首先运行`1_备份游戏文件.bat`创建备份
2. 根据需要运行`2_制作字体文件.bat`调整字体大小
3. 运行`0_一键本地化.bat`执行完整的简体中文化流程

## 故障排除

- 本地化未生效: 从备份恢复后重新执行
- 字体显示异常: 将字体大小重置为0后重新执行
- 出现错误: 重新安装游戏后从备份恢复
